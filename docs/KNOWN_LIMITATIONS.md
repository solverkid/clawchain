# ClawChain 已知限制与安全事项

**版本**: 测试网 v0.1  
**更新日期**: 2026-03-19

---

## 1. 钱包与认证安全

### 1.1 当前状态（测试网）
- **纯地址注册**：矿工注册只需提供 bech32 地址，无需私钥签名验证
- **REST API 无鉴权**：`/clawchain/challenge/submit` 等 API 不验证请求方身份
- **冒名提交风险**：任何人知道矿工地址即可代其提交答案（测试网可接受）

### 1.2 主网前必须解决
- [ ] 答案提交需包含矿工私钥签名（类似 Cosmos SDK tx 签名流程）
- [ ] REST API 添加 nonce 防重放攻击
- [ ] 矿工注册需要链上质押交易（而非仅写 KV store）
- [ ] 钱包密钥管理 — BIP-39 助记词 + AES-256 本地加密存储

---

## 2. 共识与验证

### 2.1 当前状态
- **单验证者模式**：测试网只有 1 个验证者（创始节点）
- **REST API 直接写 store**：REST handler 直接读写 KVStore，绕过 tx 流程
- **无 VRF 随机**：挑战分配使用 `math/rand` + 区块哈希作为种子，不是真正的 VRF

### 2.2 主网前必须解决
- [ ] 多验证者支持（最少 4 个）
- [ ] 所有状态变更通过 SDK Message/Tx 流程（不绕过共识）
- [ ] 引入 VRF（可验证随机函数）用于挑战分配
- [ ] 交易手续费机制（目前 gas = 0）

---

## 3. 经济模型

### 3.1 当前状态
- **奖励池有限**：当前测试网创世 mint 了 1,000,000,000 uclaw（1,000 CLAW）到 challenge 模块账户（仅供测试，主网按 50 CLAW/epoch 减半曲线释放）
- **质押未链上执行**：`min_stake` 参数存在但质押通过链下记录，未真正锁定代币
- **无 Slash 实际执行**：Slash 逻辑写在代码中但不通过 bankKeeper 真正扣款

### 3.2 主网前必须解决
- [ ] 主网奖励通过 50 CLAW/epoch 减半曲线释放（100% Fair Launch，21,000,000 CLAW 全部通过挖矿分发）
- [ ] 质押需要通过 staking 模块真正锁定代币
- [ ] Slash 需要通过 bankKeeper.SendCoinsFromModuleToModule 执行
- [ ] 解质押冷却期（7天）需要通过队列实际管理

## 4. 性能与可扩展性

### 4.1 当前状态
- **KV store JSON 序列化**：所有矿工和挑战数据以 JSON 存储在 IAVL store 中
- **线性遍历**：统计 API（`/clawchain/stats`）全量遍历所有挑战和矿工
- **无分页**：API 返回所有数据，无 limit/offset

### 4.2 生产环境需优化
- [ ] 使用 Protobuf 序列化替代 JSON
- [ ] 添加索引（按 epoch、status 等）
- [ ] API 添加分页支持
- [ ] 考虑状态裁剪（pruning old challenges）

---

## 5. 测试网 vs 主网关键区别

| 维度 | 测试网（当前） | 主网（目标） |
|------|--------------|------------|
| 验证者数量 | 1 | ≥ 4，目标 10-100 |
| 出块时间 | 2s（测试）/ 6s（配置） | 6s |
| 代币价值 | 无 | 市场定价 |
| API 安全 | 无鉴权 | 签名验证 |
| 质押 | 链下记录 | 链上锁定 |
| 挑战过期 | DEV 模式跳过 | 200 blocks（20 分钟） |
| 奖励 | 立即到账 | 通过 tx 结算 |
| 数据持久化 | 可随时重置 | 永久链上 |

---

## 6. 安全最佳实践（测试网用户）

1. **不要使用主网钱包地址**参与测试网
2. **测试网代币无价值**，不要花钱购买
3. **不要暴露 API 端口**到公网（默认 localhost）
4. **定期备份**测试数据和日志
5. **报告 Bug**到 GitHub Issues
